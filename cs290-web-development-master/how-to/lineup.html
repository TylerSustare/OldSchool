Get Your Roster

In this section, we will go over some code to get a Gym-Leader like lineup with a common theme. Now just like the last page, you can customize this with anotehr type if you like, but for demo we will be going with a demo type of: "Dragon". 

        <?php
          $id = 1; // start the counter at the first Pokemon
          while ($id <= 722){ // 722 pokemon to go through
            $data = file_get_contents("http://pokeapi.co/api/v1/pokemon/".$id)
            $rData = json_decode($data, true);
            if($rData['types']=="Dragon" && $rData['attack']>=50){
              echo("".$rData['name']."<br/>")
            }
            $id++;
          }
        ?>

Some things have changed, but we are getting into more complexities of the API. We are able to keep getting pokemon JSON objects back when we make calls to the API, and teh simple while loop allows us to do that. Now we need to use the #id variable to keep track of the pokemon count up to 722 of them. We still parse the data the same way, but now use different keys to get the "Type" and "Attack" statistics. We are only interested in strong, Dragon type pokemon for this list so we only echo back out the ones that fit our parameters.
